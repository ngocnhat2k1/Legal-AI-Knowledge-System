# Agent Memory Index

This file is the navigation map for durable agent memory in **Customs Assistant**. Start here, then follow the linked source-of-truth notes needed for the current task.

## Read This First

Customs Assistant is an internal tool for customs declaration work at a Vietnamese logistics company. Two rules govern almost every decision in this repository:

1. **Tariff rates are looked up by exact key, never generated by an LLM.** Semantic search over a tariff table returns the most similar-looking row, which in a tariff table is usually the wrong row. See [Business Rules](business-rules.md) and [ADR: No LLM On Tariff Numbers](architecture-decisions/2026-07-17-no-llm-on-tariff-numbers.md).
2. **HS codes are offered as top-3 candidates with verbatim legal evidence, never asserted.** Independent benchmarks put autonomous LLM accuracy at 29–47% at 10-digit against a 95% human baseline; explainable top-3 with evidence reaches 93.9%. The gap is the output contract, not model capability. See [ADR: HS Candidates Not Answers](architecture-decisions/2026-07-17-hs-candidates-not-answers.md).

If a change would weaken either, stop and raise it with the owner.

## Provenance And Staleness

The domain and data-source notes were written on 2026-07-17 from twelve research reports that fetched live sources, including three adversarial verification passes. Where two reports contradicted each other — notably on whether the customs.gov.vn tariff API is reachable and captcha-free — **both are recorded and the conflict is marked unresolved** rather than silently resolved. See `Open Conflicts` in [Data Sources](concepts/data-sources.md).

Vietnamese law changes monthly, and this corpus is unusually volatile: a decree can be issued, take effect the same day, be gazetted two weeks later, and expire eight weeks after that. **Every factual claim in these notes carries a verification date.** Treat any claim older than a few months as a lead, not a fact, and re-verify before relying on it.
